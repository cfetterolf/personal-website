Endeavor

Confronted with the difficulty of managing numerous job applications, I created Endeavor to facilitate and expedite this often tedious process. Endeavor is a centralized platform that performs many functions, primarily:

  • Timeline - Track job applications

  • Tasks - Record tasks to complete

  • Contacts - Organize networking contacts

  • Discover - Given name and company, find valid email addresses

  • Connect - Send templated emails to existing contacts

Much thanks to Connor Hanify for creating the Timeline and Tasks sections!

Flock

Flock was conceived as a way to let Middlebury students find and share rides more effectively. The web application allows students to log in with their Middlebury email, and provides five core features:

  • Find a Ride - View existing rides offered by other users and reserve a seat

  • Post a Ride - Offer a ride to other users by specifying origin, destination, seats available, and date

  • Profile - See your current offers, requests reservations, and account information

  • Interactive Map - Display the currently selected ride as a route on a searchable map

  • Notifications - Recieve email and text notifications when booking, requesting, or offering a ride

I worked on Flock as part of an 8-person team for my software development course in Spring 2018. Throughout the 6-week development period, our team followed Agile practices such as 2-week sprints, stand-up meetings, and pair programming. My primary contributions to the project were:

  • UI design and implementation for the Rides page (pictured above) and Login page

  • Profile sections where users could view their current offers, requests, reservations, and account information

  • Google Maps API implementation, pair-programmed with Kyle Schlanger

  • Node.js endpoint that uses Nodemailer to send notification emails

Check it out at https://secret-plains-24328.herokuapp.com (only Middlebury students can create an account, unfortunately)

Group Scheduler

Every semester, Middlebury's Riddim Dance Troupe faces a scheduling dilemma. The director receives choreographers’ availability and must schedule rehearsals around those constraints, which is often a tedious and time-consuming task.

The goal of this project was to implement an algorithm that takes three factors into account—choreographers' availability, dancers’ availability, and dancers’ preferences for specic choreographers—and outputs an optimal schedule for all parties involved.

The algorithm, which I wrote in Python, ran inside of a Django web application. A variation on the Hungarian Algorithm, it tackled a more complex version of the assignment problem by finding an optimal matching based on the three conditions mentioned above.
